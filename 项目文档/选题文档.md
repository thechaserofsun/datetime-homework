# 可疑进程行为分析与风险评分程序 — 选题文档

## 一、选题题目

**基于 Windows 的可疑进程行为分析与风险评分系统**

> 研究对象：可疑进程；工作方式：行为采集 + 规则匹配 + 加权评分；应用场景：Windows 主机安全巡检

---

## 二、问题背景

Windows 环境下恶意软件常以合法进程名伪装运行，或通过异常行为（非预期外连、可疑子进程派生、敏感文件访问等）实施攻击。传统杀软依赖特征库，对无特征的新型威胁缺乏感知能力。本系统从**行为侧**出发，通过采集进程的多维度行为特征，基于规则加权评分模型识别可疑进程，将主机安全风险转化为"可见、可测、可感"的量化结果。

---

## 三、设计目标

1. **行为采集**：自动采集进程的树状关系、网络连接、文件操作、DLL 加载等多维行为数据
2. **规则匹配**：定义覆盖常见可疑行为的规则集，支持规则命中判定
3. **风险评分**：基于规则加权打分模型，对每个进程输出风险分值与等级
4. **风险展示**：以 Web 界面可视化呈现风险进程、证据链与等级分布
5. **定时巡检**：支持后台定时扫描，记录历史快照，展示风险趋势变化

---

## 四、技术选型

| 层级 | 技术 | 说明 |
|------|------|------|
| 后端框架 | Python + Flask | 轻量 Web 框架，提供 RESTful API |
| 前端 | 原生 HTML/CSS/JS | 无需构建工具，直接 `.html` 文件 |
| 进程采集 | psutil | 跨平台进程信息库，获取进程树、连接、文件句柄等 |
| 评分模型 | 规则加权打分 | 每条规则设权重与分值，命中累加 |
| 数据库 | MySQL | 持久化进程快照、扫描记录、风险评分结果 |
| 定时任务 | APScheduler | Flask 内嵌调度器，支持定时/周期扫描 |

## 六、核心模块说明

### 1. 进程采集模块 (`scanner.py`)

基于 psutil 采集以下维度：

| 维度 | 采集内容 | psutil 接口 |
|------|----------|-------------|
| 基本信息 | PID、进程名、路径、命令行、用户 | `Process.info` |
| 进程树 | 父子进程关系 | `Process.parent() / children()` |
| 网络连接 | 远端 IP/端口、连接状态 | `Process.connections()` |
| 文件操作 | 打开的文件句柄 | `Process.open_files()` |
| DLL 加载 | 加载的动态链接库 | `Process.memory_maps()` |
| 资源占用 | CPU、内存使用率 | `Process.cpu_percent() / memory_info()` |

### 2. 规则引擎与评分模块 (`ruler.py`)

**规则分类与权重示例：**

| 规则类别 | 规则示例 | 权重 | 单项分值 |
|----------|----------|------|----------|
| 伪装检测 | 进程名模仿系统进程但路径不在系统目录 | 高 | 30 |
| 网络异常 | 非浏览器进程发起外连 | 中 | 20 |
| 进程关系 | 系统进程派生非预期子进程 | 高 | 25 |
| 文件访问 | 访问敏感目录（用户文档、凭据目录） | 中 | 20 |
| 无签名/路径 | 进程路径为空或不可读 | 中 | 15 |
| 资源异常 | 长期高 CPU 占用 | 低 | 10 |

**评分逻辑：**
- 进程总分 = Σ(命中规则分值 × 权重系数)
- 风险等级映射：0–20 低风险 | 21–50 中风险 | 51–80 高风险 | 81+ 极高风险

### 3. API 路由模块 (`app.py`)

| 路由 | 方法 | 功能 |
|------|------|------|
| `/api/scan` | POST | 触发一次手动扫描 |
| `/api/processes` | GET | 获取最新扫描的进程列表及评分 |
| `/api/process/<pid>` | GET | 获取单个进程的详细行为与命中规则 |
| `/api/rules` | GET | 获取规则列表 |
| `/api/rules` | POST | 新增自定义规则 |
| `/api/history` | GET | 获取历史扫描记录概要 |
| `/api/trend` | GET | 获取风险趋势数据（用于折线图） |
| `/api/schedule` | POST | 设置定时巡检间隔 |

### 4. 定时巡检模块

基于 APScheduler，启动时默认加载调度任务：
- 支持自定义巡检间隔（如每 5 分钟 / 30 分钟 / 1 小时）
- 每次巡检生成一份快照存入 MySQL
- 前端可查看历史趋势

## 七、数据库设计

### 表结构概览

**scan_record（扫描记录表）**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT AUTO_INCREMENT | 主键 |
| scan_time | DATETIME | 扫描时间 |
| total_processes | INT | 本次扫描进程总数 |
| high_risk_count | INT | 高风险进程数 |
| trigger_type | ENUM('manual','scheduled') | 触发方式 |

**process_snapshot（进程快照表）**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT AUTO_INCREMENT | 主键 |
| scan_id | INT | 关联扫描记录 |
| pid | INT | 进程 PID |
| name | VARCHAR(100) | 进程名 |
| exe_path | TEXT | 可执行路径 |
| cmdline | TEXT | 命令行 |
| username | VARCHAR(50) | 运行用户 |
| parent_pid | INT | 父进程 PID |
| risk_score | INT | 风险分值 |
| risk_level | ENUM('low','medium','high','critical') | 风险等级 |

**rule_hit（规则命中表）**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT AUTO_INCREMENT | 主键 |
| snapshot_id | INT | 关联进程快照 |
| rule_id | INT | 关联规则 |
| detail | TEXT | 命中详情（如异常远端 IP） |

**rule（规则表）**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | INT AUTO_INCREMENT | 主键 |
| category | VARCHAR(50) | 规则类别 |
| name | VARCHAR(100) | 规则名称 |
| description | TEXT | 规则描述 |
| weight | FLOAT | 权重系数 |
| score | INT | 单项分值 |
| is_builtin | BOOLEAN | 是否内置规则 |

## 九、风险展示规划（对应报告"五小节法"）

系统将为每个高风险/极高风险进程自动生成风险报告卡片，包含：

1. **风险对象**：该进程名、PID、所属类别（伪装/外连/异常派生等）
2. **触发条件**：命中的具体规则及判定逻辑
3. **风险证据**：异常远端 IP 列表、可疑路径、异常父子关系截图
4. **影响分析**：可能导致的后果（数据泄露、权限提升、持久化等）
5. **展示结论**：风险等级判定 + 理由
