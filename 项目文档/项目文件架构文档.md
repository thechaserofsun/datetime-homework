## 各目录/文件职责详述

### 1. `项目文档/` — 需求与设计

| 文件 | 说明 |
|------|------|
| 平时作业要求.md | 老师下发的作业原始要求 |
| 平时作业说明.md | 老师下发的评分细则与撰写指南 |
| 选题文档.md | 本项目的选题分析、技术选型、架构设计 |

> 此目录仅存放文档，不参与代码运行。

---

### 2. `src/` — 后端源码

| 文件 | 职责 | 对外依赖 |
|------|------|----------|
| **app.py** | Flask 应用工厂：初始化 app、注册蓝图、挂载定时调度、启动服务 | flask, flask_sqlalchemy, apscheduler |
| **scanner.py** | 进程采集：调用 psutil 获取进程基本信息、进程树、网络连接、文件句柄、内存映射、资源占用，返回结构化字典 | psutil |
| **ruler.py** | 规则引擎：加载规则集，对采集结果逐条匹配，计算加权分值，输出风险等级与命中详情 | 无外部依赖 |
| **models.py** | 数据模型：定义 scan_record、process_snapshot、rule_hit、rule 四张表的 SQLAlchemy Model | flask_sqlalchemy |
| **scheduler.py** | 定时调度：封装 APScheduler，提供启动/停止/修改间隔接口，每次触发调用 scanner + ruler + 入库 | apscheduler |
| **config.py** | 配置中心：MySQL 连接串、扫描间隔默认值、风险等级阈值、日志级别等 | 无外部依赖 |

**模块调用关系：**

```
app.py
 ├── scanner.py    (采集进程数据)
 ├── ruler.py      (规则匹配 + 评分)
 ├── models.py     (数据库读写)
 ├── scheduler.py  (定时巡检，内部调用 scanner + ruler + models)
 └── config.py     (读取配置)
```

---

### 3. `sql/` — 数据库初始化脚本

| 文件 | 说明 |
|------|------|
| **init.sql** | 包含 CREATE TABLE 建表语句 + INSERT 内置规则初始数据。首次部署时手动执行一次，也可由 `app.py --init` 自动调用 |

---

### 4. `templates/` — 前端页面

| 文件 | 页面功能 | 核心 UI 元素 |
|------|----------|-------------|
| **index.html** | 仪表盘 | 风险等级分布饼图、进程总数/高风险数统计卡片、最近一次扫描摘要、风险趋势折线图、手动扫描按钮 |
| **processes.html** | 进程列表 | 可排序表格（PID/名称/路径/评分/等级）、风险等级筛选下拉框、关键词搜索框、点击行跳转详情 |
| **detail.html** | 进程详情 | 进程基本信息卡片、进程树可视化、网络连接列表、DLL 列表、命中规则及详情、风险报告卡片（五小节法） |
| **rules.html** | 规则管理 | 内置规则表格（类别/名称/权重/分值）、自定义规则添加表单、启用/禁用开关 |
| **history.html** | 历史记录 | 扫描记录时间线表格、选定两次扫描的风险对比、全时段趋势折线图 |

> 所有页面通过 `fetch()` 调用后端 REST API，无模板渲染逻辑（Jinja 仅用于页面骨架）。

---

### 5. `static/` — 静态资源

| 文件 | 职责 |
|------|------|
| **css/style.css** | 全局样式：配色、表格、卡片、布局、响应式适配 |
| **js/dashboard.js** | 仪表盘专用：调用 `/api/trend` 绘制折线图，调用 `/api/processes` 绘制饼图，处理手动扫描按钮 |
| **js/processes.js** | 进程列表专用：调用 `/api/processes` 渲染表格，处理筛选、搜索、分页 |
| **js/detail.js** | 详情页专用：调用 `/api/process/<pid>` 渲染行为数据与命中规则 |
| **js/rules.js** | 规则页专用：调用 `/api/rules` 渲染规则表，处理新增规则表单提交 |
| **js/history.js** | 历史页专用：调用 `/api/history` 和 `/api/trend` 渲染记录与趋势图 |

> 图表库使用 Chart.js（CDN 引入，不放入 static）。

---

### 6. `logs/` — 运行日志

程序运行时自动生成的日志文件，不纳入版本管理。

| 日志内容 | 说明 |
|----------|------|
| 扫描日志 | 每次扫描的时间、进程数、耗时、异常 |
| 异常日志 | psutil 访问被拒、数据库连接失败等 |
