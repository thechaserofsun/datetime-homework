# 实现计划

## 阶段总览

| 阶段 | 目标 | 产出 |
|------|------|------|
| 一 | Flask 骨架 + 数据库初始化 + 基础 API | 项目可启动，数据库可建表，API 可访问 |

## 阶段二：进程采集模块

**目标**：基于 psutil 完成进程全维度行为采集，返回结构化数据。

### 任务清单

- [ ] 编写 `src/scanner.py`
  - `scan_all_processes()` — 采集全部进程列表
  - `scan_single_process(pid)` — 采集单个进程详情
- [ ] 采集维度实现
  - 基本信息：PID、名称、路径、命令行、用户
  - 进程树：父进程 PID、子进程列表
  - 网络连接：远端 IP、远端端口、连接状态、协议
  - 文件句柄：打开的文件路径列表
  - DLL/内存映射：加载的 .dll 路径列表
  - 资源占用：CPU 百分比、内存占用（RSS）
- [ ] 异常处理：psutil.AccessDenied / NoSuchProcess 时跳过并记录日志
- [ ] 编写采集结果的数据结构（字典/NamedTuple），供评分模块直接消费
- [ ] 对接 API：`/api/scan` 调用 `scan_all_processes()`，将结果写入数据库

### 阶段完成标志

- 调用 `/api/scan` 后，MySQL 的 `process_snapshot` 表中出现当前系统全部进程记录
- 访问 `/api/processes` 返回含进程基本信息的 JSON 列表
- 访问 `/api/process/<pid>` 返回该进程全部行为维度数据

## 阶段三：规则引擎 + 加权评分

**目标**：对采集数据逐条匹配规则，输出风险分值、等级和命中详情。

### 任务清单

- [ ] 编写 `src/ruler.py`
  - `load_rules()` — 从数据库加载启用的规则
  - `evaluate(process_data, rules)` — 对单个进程执行规则匹配，返回命中列表
  - `calculate_score(hits)` — 加权累加计算总分
  - `determine_level(score)` — 分值映射为风险等级（low/medium/high/critical）
- [ ] 实现内置规则匹配逻辑（至少覆盖以下类别）
  - 伪装检测：进程名含系统关键词（svchost、lsass、csrss）但路径不在 System32
  - 网络异常：非浏览器进程（排除 chrome、msedge、firefox 等）发起外连
  - 进程关系异常：explorer.exe 派生 cmd/powershell、系统进程派生非预期子进程
  - 路径异常：进程可执行路径为空或不存在
  - 文件访问异常：访问用户凭据目录、浏览器数据目录
  - 资源异常：CPU 占用超过阈值持续运行
- [ ] 命中详情记录：每条命中写入 `rule_hit` 表，detail 字段记录具体证据（如异常远端 IP）
- [ ] 对接 API：扫描流程变为 采集 → 评分 → 入库，`/api/processes` 返回含评分的结果

### 阶段完成标志

- 执行扫描后，每个进程均有 risk_score 和 risk_level
- `/api/processes` 返回的数据包含评分和等级
- `/api/process/<pid>` 返回该进程命中的规则列表及详情

## 阶段六：风险展示完善 + 整体联调

**目标**：风险报告符合"五小节法"，整体功能稳定。

- [ ] 边界处理

  - API 参数校验与错误返回
- [ ] 日志完善：`logs/` 下按日期生成扫描日志与异常日志
- [ ] 整体联调：完整走通 手动扫描 → 查看列表 → 查看详情 → 查看风险报告 → 查看历史趋势 的全流程
